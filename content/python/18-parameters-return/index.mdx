# Parameters and Return

<StoryBox character="python_pete">
  Your functions are working great, but they're a bit... predictable. They do the same thing every time!

  Imagine if your phone calculator always added 2 + 2. Not very useful, right? You need to be able to give it DIFFERENT numbers!

  That's what parameters and return values are for. They make functions POWERFUL and FLEXIBLE!
</StoryBox>

## What Are Parameters?

**Parameters** are like function inputs - information you pass INTO the function:

<TerminalDemo>
{`# Function WITHOUT parameters (always the same)
def greet():
    print("Hello, friend!")

greet()  # Output: Hello, friend!
greet()  # Output: Hello, friend!

# Function WITH parameters (flexible!)
def greet(name):
    print(f"Hello, {name}!")

greet("Alice")    # Output: Hello, Alice!
greet("Bob")      # Output: Hello, Bob!
greet("Charlie")  # Output: Hello, Charlie!`}
</TerminalDemo>

See the difference? The parameter `name` lets us customize the greeting!

<FunFact>
  The terms "parameter" and "argument" are often used interchangeably, but technically: a **parameter** is the variable in the function definition, and an **argument** is the actual value you pass when calling the function!
</FunFact>

## How Parameters Work

Think of parameters as variables that get their values from the function call:

```python
def welcome(player_name):
    # player_name is the parameter
    print(f"Welcome to the game, {player_name}!")

# "Hero123" is the argument
welcome("Hero123")
```

When you call `welcome("Hero123")`, Python basically does this:
```python
player_name = "Hero123"  # Sets the parameter
print(f"Welcome to the game, {player_name}!")
```

<StoryBox character="python_pete">
  Think of parameters like blanks in a Mad Libs game! The function has blank spots (parameters), and when you call it, you fill in those blanks with actual values (arguments).

  Function definition: "Hello, _____ (name)!"
  Function call: welcome("Sarah") fills in "Sarah"!
</StoryBox>

## Multiple Parameters

Functions can have as many parameters as you need:

<TerminalDemo>
{`# Function with TWO parameters
def introduce(name, age):
    print(f"My name is {name} and I'm {age} years old.")

introduce("Luna", 12)
# Output: My name is Luna and I'm 12 years old.

# Function with THREE parameters
def create_character(name, health, level):
    print(f"Character Created!")
    print(f"Name: {name}")
    print(f"Health: {health}")
    print(f"Level: {level}")

create_character("DragonSlayer", 100, 5)
# Output:
# Character Created!
# Name: DragonSlayer
# Health: 100
# Level: 5`}
</TerminalDemo>

**Important:** When calling a function with multiple parameters, the ORDER matters!

```python
def describe_pet(animal, name):
    print(f"I have a {animal} named {name}.")

describe_pet("dog", "Buddy")    # Correct order
describe_pet("Buddy", "dog")    # Wrong order - confusing!
```

## Return Values: Getting Information BACK

Parameters let you send data IN. But what about getting data OUT? That's what `return` does!

<TerminalDemo>
{`# Function that RETURNS a value
def add_numbers(a, b):
    result = a + b
    return result

# The function gives us back the answer!
sum = add_numbers(5, 3)
print(sum)  # Output: 8

# Use it directly in other operations
total = add_numbers(10, 20) + 5
print(total)  # Output: 35

# Compare to a function that just prints
def add_and_print(a, b):
    result = a + b
    print(result)

add_and_print(5, 3)  # Prints 8
value = add_and_print(5, 3)  # Prints 8, but value is None!
print(value)  # Output: None`}
</TerminalDemo>

<StoryBox character="python_pete">
  Here's the key difference:
  - **print()** shows you something on the screen (for humans to read)
  - **return** gives a value back to the program (for code to use)

  Think of it like a vending machine:
  - You put money IN (parameters)
  - The machine RETURNS a snack (return value)
  - The machine might also display a message (print), but the important thing is you GET your snack!
</StoryBox>

## The Return Statement

When Python hits a `return` statement:
1. The function immediately stops
2. The value is sent back to where the function was called

<TerminalDemo>
{`def check_password(password):
    if password == "secret123":
        return "Access granted!"
    return "Access denied!"

result = check_password("secret123")
print(result)  # Output: Access granted!

result = check_password("wrong")
print(result)  # Output: Access denied!`}
</TerminalDemo>

## Return Stops the Function

Once `return` is executed, the function exits immediately:

<TerminalDemo>
{`def find_first_vowel(word):
    for letter in word:
        if letter in "aeiou":
            return letter  # Exits immediately when found
        print(f"Checking {letter}...")
    return "No vowels found"

result = find_first_vowel("python")
print(f"First vowel: {result}")
# Output:
# Checking p...
# Checking y...
# Checking t...
# Checking h...
# First vowel: o`}
</TerminalDemo>

## Practical Examples

### Example 1: Calculate Area
<TerminalDemo>
{`def calculate_rectangle_area(width, height):
    area = width * height
    return area

room_area = calculate_rectangle_area(10, 12)
print(f"Room area: {room_area} square meters")
# Output: Room area: 120 square meters

# Use in calculations
paint_needed = calculate_rectangle_area(10, 12) * 0.5
print(f"Paint needed: {paint_needed} liters")
# Output: Paint needed: 60.0 liters`}
</TerminalDemo>

### Example 2: Temperature Converter
<TerminalDemo>
{`def celsius_to_fahrenheit(celsius):
    fahrenheit = (celsius * 9/5) + 32
    return fahrenheit

temp_f = celsius_to_fahrenheit(25)
print(f"25°C = {temp_f}°F")
# Output: 25°C = 77.0°F

temp_f = celsius_to_fahrenheit(0)
print(f"0°C = {temp_f}°F")
# Output: 0°C = 32.0°F`}
</TerminalDemo>

### Example 3: String Formatter
<TerminalDemo>
{`def create_username(first_name, last_name):
    username = (first_name[0] + last_name).lower()
    return username

user1 = create_username("Sarah", "Johnson")
print(user1)  # Output: sjohnson

user2 = create_username("Alex", "Chen")
print(user2)  # Output: achen`}
</TerminalDemo>

<FunFact>
  Professional programmers often say functions should "do one thing and do it well." A function that calculates something should return the result, not print it. This makes the function more flexible and reusable!
</FunFact>

## Default Parameters

You can give parameters default values:

<TerminalDemo>
{`def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

print(greet("Alice"))              # Output: Hello, Alice!
print(greet("Bob", "Hi"))          # Output: Hi, Bob!
print(greet("Charlie", "Howdy"))   # Output: Howdy, Charlie!`}
</TerminalDemo>

If you don't provide the `greeting` argument, it uses "Hello" by default!

## Common Mistakes

### Mistake 1: Forgetting to Return
```python
# WRONG - Calculates but doesn't return
def add(a, b):
    result = a + b
    # Missing return!

answer = add(5, 3)
print(answer)  # Output: None

# RIGHT
def add(a, b):
    result = a + b
    return result

answer = add(5, 3)
print(answer)  # Output: 8
```

### Mistake 2: Using Print Instead of Return
```python
# WRONG - Prints instead of returning
def double(number):
    print(number * 2)

result = double(5)  # Prints 10, but result is None!

# RIGHT
def double(number):
    return number * 2

result = double(5)  # Returns 10, result is 10!
```

### Mistake 3: Wrong Number of Arguments
```python
def greet(first_name, last_name):
    return f"Hello, {first_name} {last_name}!"

greet("Alice")  # ERROR! Missing last_name
greet("Alice", "Smith", "Extra")  # ERROR! Too many arguments
greet("Alice", "Smith")  # Correct!
```
