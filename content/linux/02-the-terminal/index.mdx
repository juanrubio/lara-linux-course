## What is the Terminal?

The **Terminal** (also called the command line, shell, or console) is a text-based interface for controlling your computer. Instead of clicking on icons, you type commands.

Think of it like this:
- **Graphical Interface (GUI)**: Like pointing at things you want
- **Terminal**: Like telling someone exactly what to do with words

<FunFact>
  Before mice and touchscreens existed, the ONLY way to use computers was through text commands! The first programmers were all terminal experts.
</FunFact>

## The Anatomy of a Prompt

When you open the terminal, you see something like this:

<TerminalDemo>
{`learner@codequest:~$`}
</TerminalDemo>

Let's break it down:

| Part | Meaning |
|------|---------|
| `learner` | Your username |
| `@` | "at" |
| `codequest` | Your computer's name |
| `:` | Separator |
| `~` | Your current location (~ means home folder) |
| `$` | "Ready for your command!" |

## Your First Look at Commands

A command has this structure:

```bash
command [options] [arguments]
```

- **Command**: What action to perform
- **Options**: How to do it (usually start with `-`)
- **Arguments**: What to do it on

Example:
```bash
ls -l Documents
```
- `ls` = list files (command)
- `-l` = show details (option)
- `Documents` = which folder (argument)
