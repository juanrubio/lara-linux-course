## The Man Pages: Your Built-in Encyclopedia

Linux has a powerful built-in manual called "man pages" (short for manual pages). Every command has its own manual page!

To read a manual, type `man` followed by the command name:

<TerminalDemo>
{`learner@codequest:~$ man ls
LS(1)                            User Commands                           LS(1)

NAME
       ls - list directory contents

SYNOPSIS
       ls [OPTION]... [FILE]...

DESCRIPTION
       List  information  about  the FILEs (the current directory by default).

       -a, --all
              do not ignore entries starting with .

       -l     use a long listing format
       ...`}
</TerminalDemo>

<FunFact>
  The man pages have been around since 1971! They're older than most people's parents. Every Linux system has them built in - no internet needed!
</FunFact>

### Navigating Man Pages

When you're inside a man page:

| Key | Action |
|-----|--------|
| `Space` or `Page Down` | Scroll down one page |
| `b` or `Page Up` | Scroll up one page |
| `↑` `↓` | Scroll line by line |
| `/word` | Search for "word" |
| `n` | Jump to next search result |
| `q` | Quit (very important!) |

<StoryBox character="pixel_the_penguin">
  Don't panic if you get stuck in a man page! Just press `q` to quit and return to your command prompt. It's like the exit door in a library!
</StoryBox>

## Quick Help with --help

Sometimes you just need a quick reminder, not the whole manual. Most commands support `--help`:

<TerminalDemo>
{`learner@codequest:~$ ls --help
Usage: ls [OPTION]... [FILE]...
List information about the FILEs (the current directory by default).

Mandatory arguments to long options are mandatory for short options too.
  -a, --all                  do not ignore entries starting with .
  -l                         use a long listing format
  ...`}
</TerminalDemo>

This gives you a shorter, quicker reference!

## Which Should You Use?

| Situation | Use |
|-----------|-----|
| Need detailed explanation | `man command` |
| Quick reminder of options | `command --help` |
| Learning a new command | `man command` first |

## Bonus: The whatis Command

Want a one-line description of a command? Use `whatis`:

<TerminalDemo>
{`learner@codequest:~$ whatis ls
ls (1)               - list directory contents

learner@codequest:~$ whatis pwd
pwd (1)              - print name of current/working directory`}
</TerminalDemo>

Super quick way to remember what a command does!
