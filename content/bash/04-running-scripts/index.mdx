## Understanding File Permissions

In Linux, every file has **permissions** that control:
1. Who can READ the file
2. Who can WRITE (modify) the file
3. Who can EXECUTE (run) the file

Let's look at a file's permissions:

<TerminalDemo>
{`$ ls -l hello.sh
-rw-r--r-- 1 pi pi 45 Dec 19 10:30 hello.sh`}
</TerminalDemo>

Let's break down those weird letters:


<FunFact>

</FunFact>

## Making Scripts Executable

To make a script executable, we use the `chmod` command (change mode):

<TerminalDemo>
{`$ chmod +x hello.sh`}
</TerminalDemo>

Now check the permissions again:

<TerminalDemo>
{`$ ls -l hello.sh
-rwxr-xr-x 1 pi pi 45 Dec 19 10:30 hello.sh`}
</TerminalDemo>

See those `x` letters? Now everyone can execute the file!

## Running Executable Scripts

Now you can run your script with `./`:

<TerminalDemo>
{`$ ./hello.sh
Hello, World!
This is my first shell script!
I'm learning Bash scripting!`}
</TerminalDemo>

It works! The `./` means "run the file in the current directory."

<StoryBox character="bash_betty">
  Why do we need `./`?

  Because Linux doesn't automatically look in the current directory for programs. The `./` tells it "run the script that's RIGHT HERE!"

  Think of `./` as pointing and saying "this one, right here!"
</StoryBox>

## Understanding chmod

The `chmod` command has several options:

```bash
# Add execute permission
chmod +x script.sh

# Remove execute permission
chmod -x script.sh

# Make it executable only for you
chmod u+x script.sh

# Make it executable for everyone
chmod a+x script.sh
```

Let's break down `chmod +x`:
- `chmod` - Change mode (permissions)
- `+` - Add permission
- `x` - Execute permission

<FunFact>
  chmod can also use numbers! `chmod 755 script.sh` does the same as `chmod +x`. The numbers represent different permission combinations!
</FunFact>

## Where Do Programs Live?

You might wonder - why don't we need `./` for commands like `ls` or `cd`?

Because they're stored in special directories that Linux always checks. To see these directories:

<TerminalDemo>
{`$ echo $PATH
/usr/local/bin:/usr/bin:/bin`}
</TerminalDemo>

These directories are in your `PATH` - Linux automatically looks there for programs!

Want to run your script from anywhere? Move it to one of these directories (we'll learn this later)!

## Troubleshooting Common Issues

### Issue 1: Command not found
```bash
$ hello.sh
bash: hello.sh: command not found
```
**Solution:** Use `./hello.sh` instead!

### Issue 2: Permission denied
```bash
$ ./hello.sh
bash: ./hello.sh: Permission denied
```
**Solution:** Run `chmod +x hello.sh` first!

### Issue 3: Bad interpreter
```bash
$ ./hello.sh
bash: ./hello.sh: bad interpreter: No such file or directory
```
**Solution:** Check your shebang line - it should be `#!/bin/bash`

## Best Practices for Executable Scripts

Follow these professional tips:

1. **Always start with a shebang**
   ```bash
   #!/bin/bash
   ```

2. **Add comments at the top**
   ```bash
   #!/bin/bash
   # Script: backup.sh
   # Purpose: Backup important files
   # Author: Your Name
   # Date: 2024-01-15
   ```

3. **Make scripts executable right after creating them**
   ```bash
   chmod +x script.sh
   ```

4. **Use descriptive names**
   - âœ… `backup_photos.sh`
   - âœ… `system_check.sh`
   - âŒ `script1.sh`
