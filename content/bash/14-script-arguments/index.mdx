# Script Arguments

<StoryBox character="bash_betty">
  Welcome back, script architect! Your scripts are looking good and working with files. But they all do the SAME thing every time. Boring!

  What if your script could be different each time you run it? What if YOU could tell the script what to do when you run it? That's the power of ARGUMENTS!
</StoryBox>

## What Are Script Arguments?

Arguments are like giving instructions to your script when you run it. Just like commands take arguments!

<TerminalDemo>
{`# Commands use arguments all the time:
ls -la /home
# ls is the command
# -la and /home are ARGUMENTS

# Your scripts can do the same!
./greet.sh Alice
# ./greet.sh is your script
# Alice is an ARGUMENT`}
</TerminalDemo>

<StoryBox character="bash_betty">
  Think of arguments like customizing a pizza order:
  - The pizza shop (your script) is always there
  - But YOU tell them what toppings you want (arguments)
  - Same shop, different pizza every time!

  Same script, different behavior every time!
</StoryBox>

## Positional Parameters

Bash uses special variables to store arguments:

| Variable | What It Contains |
|----------|------------------|
| `$0` | The script name itself |
| `$1` | First argument |
| `$2` | Second argument |
| `$3` | Third argument |
| ... | And so on... |

<TerminalDemo>
{`#!/bin/bash
# Save as greet.sh

echo "Script name: $0"
echo "First argument: $1"
echo "Second argument: $2"

# Run it:
# ./greet.sh Alice Bob
#
# Output:
# Script name: ./greet.sh
# First argument: Alice
# Second argument: Bob`}
</TerminalDemo>

## Your First Argument Script

Let's create a greeting script that says hello to whoever you specify:

<TerminalDemo>
{`#!/bin/bash
# greet.sh - Personalized greeting

echo "Hello, $1!"
echo "Welcome to bash scripting!"

# Run it:
# ./greet.sh Alice
# Output: Hello, Alice!
#         Welcome to bash scripting!

# ./greet.sh Bob
# Output: Hello, Bob!
#         Welcome to bash scripting!`}
</TerminalDemo>

<FunFact>
  You can have as many arguments as you want! There's no practical limit. Scripts can accept $1, $2, $3... all the way to $999 and beyond!
</FunFact>

## Special Variables

Bash has some SUPER useful special variables for working with arguments:

<TerminalDemo>
{`#!/bin/bash

echo "Script name: $0"
echo "Number of arguments: $#"
echo "All arguments: $@"
echo "All arguments (as one): $*"
echo "Process ID: $$"
echo "Last command exit status: $?"`}
</TerminalDemo>

| Variable | What It Does |
|----------|--------------|
| `$#` | Count how many arguments |
| `$@` | All arguments as separate words |
| `$*` | All arguments as one string |
| `$?` | Exit status of last command |
| `$$` | Process ID of the script |

<StoryBox character="bash_betty">
  The `$#` is SUPER useful! You can check if someone forgot to provide arguments:

  "Did you give me 3 arguments? No? Then I'll show an error message!"

  Professional scripts ALWAYS check if they got the right number of arguments!
</StoryBox>

## Checking Argument Count

Always check if you got the arguments you need:

<TerminalDemo>
{`#!/bin/bash
# greet_safe.sh - Checks for arguments

if [ $# -eq 0 ]; then
    echo "Error: Please provide a name!"
    echo "Usage: $0 <name>"
    exit 1
fi

echo "Hello, $1!"

# Run without argument:
# ./greet_safe.sh
# Output: Error: Please provide a name!
#         Usage: ./greet_safe.sh <name>

# Run with argument:
# ./greet_safe.sh Alice
# Output: Hello, Alice!`}
</TerminalDemo>

## The Right Way to Handle Arguments

Here's a professional template for argument handling:

<TerminalDemo>
{`#!/bin/bash

# Check if we got exactly 2 arguments
if [ $# -ne 2 ]; then
    echo "Error: Wrong number of arguments"
    echo "Usage: $0 <name> <age>"
    echo "Example: $0 Alice 25"
    exit 1
fi

# Now safe to use the arguments
NAME=$1
AGE=$2

echo "Hello, $NAME!"
echo "You are $AGE years old."`}
</TerminalDemo>

<FunFact>
  The `exit 1` is important! When a script exits with 0, it means "success". Any other number (1, 2, 3...) means "error". This helps other scripts or systems know if your script worked or failed!
</FunFact>

## Looping Through All Arguments

Sometimes you don't know how many arguments you'll get. Use `$@` to loop through them all:

<TerminalDemo>
{`#!/bin/bash
# greet_all.sh - Greet everyone!

if [ $# -eq 0 ]; then
    echo "Error: Please provide at least one name"
    exit 1
fi

echo "Greeting everyone..."
echo ""

for name in "$@"; do
    echo "Hello, $name!"
done

# Run it:
# ./greet_all.sh Alice Bob Charlie
#
# Output:
# Greeting everyone...
#
# Hello, Alice!
# Hello, Bob!
# Hello, Charlie!`}
</TerminalDemo>

## Arguments vs Variables

What's the difference?

<TerminalDemo>
{`#!/bin/bash

# This is a variable YOU create
MY_VAR="Hello"

# This is an argument the USER provides
USER_INPUT=$1

echo "Variable: $MY_VAR"
echo "Argument: $USER_INPUT"

# Variables stay the same every time
# Arguments can be different every time!`}
</TerminalDemo>

## Shifting Arguments

You can "shift" arguments to process them one at a time:

<TerminalDemo>
{`#!/bin/bash

echo "First argument: $1"

shift  # Move all arguments left by one

echo "First argument after shift: $1"

# Run: ./script.sh A B C
# Output:
# First argument: A
# First argument after shift: B`}
</TerminalDemo>

<StoryBox character="bash_betty">
  Shifting is like moving through a line:
  - First person in line gets served
  - They leave
  - Next person moves to the front
  - Repeat!

  It's useful when processing arguments one by one in a loop!
</StoryBox>

## Default Values for Arguments

What if an argument is optional? Provide a default!

<TerminalDemo>
{`#!/bin/bash

# Use argument if provided, otherwise use default
NAME=${1:-"Guest"}
AGE=${2:-"unknown"}

echo "Hello, $NAME!"
echo "Age: $AGE"

# Run without arguments:
# ./script.sh
# Output: Hello, Guest!
#         Age: unknown

# Run with arguments:
# ./script.sh Alice 25
# Output: Hello, Alice!
#         Age: 25`}
</TerminalDemo>

The syntax `${VAR:-default}` means: "Use $VAR if it exists, otherwise use default"

## Real-World Example: File Search Script

<TerminalDemo>
{`#!/bin/bash
# search.sh - Search for files

# Check arguments
if [ $# -lt 1 ]; then
    echo "Error: Please provide a search term"
    echo "Usage: $0 <search_term> [directory]"
    exit 1
fi

SEARCH=$1
DIR=${2:-.}  # Use second argument or current directory

echo "Searching for '$SEARCH' in $DIR..."
echo ""

# Perform search
find "$DIR" -name "*$SEARCH*" -type f

# Run it:
# ./search.sh "data" /home/user
# Searches for files containing "data" in /home/user

# ./search.sh "data"
# Searches for files containing "data" in current directory`}
</TerminalDemo>

## Complete Example: Copy Multiple Files

<TerminalDemo>
{`#!/bin/bash
# multicopy.sh - Copy multiple files to a directory

# Need at least 2 arguments (source and destination)
if [ $# -lt 2 ]; then
    echo "Error: Not enough arguments"
    echo "Usage: $0 <file1> [file2] [...] <destination>"
    exit 1
fi

# Get all arguments
ARGS=($@)

# Last argument is the destination
DEST="${ARGS[-1]}"

# All other arguments are source files
SOURCES=("${ARGS[@]:0:$#-1}")

# Check if destination is a directory
if [ ! -d "$DEST" ]; then
    echo "Error: $DEST is not a directory"
    exit 1
fi

# Copy each file
echo "Copying files to $DEST..."
for file in "${SOURCES[@]}"; do
    if [ -f "$file" ]; then
        cp "$file" "$DEST"
        echo "✓ Copied $file"
    else
        echo "✗ Skipped $file (not found)"
    fi
done

echo "Done!"

# Run it:
# ./multicopy.sh file1.txt file2.txt file3.txt ~/backups/`}
</TerminalDemo>
